# 演出ルーレット（visual / probability split roulette）

盤面の「見た目の大きさ」と「実際の当選率」を**別々に設定できる**Webルーレットです。1ファイル完結（`index.html`）で、サーバー不要・依存ライブラリなしで動きます。

## 特長

- **見た目と当選率を分離**：各セグメントに「見た目サイズ（盤面の角度）」と「当選率（重み）」を独立して設定。『大きく見えるのに実は激レア』が作れます。
- **抽選スケジュール**：回ごとに `操作（当選率）` か `表示通り（見た目どおり）` を切替。初期値は **1回目＝操作 / 2回目＝表示通り**。回数はページを開くたびに1回目へ戻ります。
- **管理画面 / プレイヤー画面の分離**：プレイヤー側にはホイールとSPINだけが表示され、セグメント設定や当選率は見えません。
- **当たり演出**：着地スライスの発光＋効果音（Web Audio で合成。当選率が低いほど豪華に）。
- **検証（モンテカルロ）**：N回テストで、実測の出現率が見た目ではなく当選率に収束することを確認できます。

## 使い方

- `index.html` をそのまま開くと**管理画面**。色・ラベル・見た目サイズ・当選率・スケジュールを設定します。
- 「**プレイヤー画面をプレビュー**」で、配布時に見える画面を確認できます。
- 「**公開リンクをコピー**」で `#play=…` 付きURLを発行 → スピンする人へ配布。開いた人には設定・確率は見えません。
- `Space` キーでもスピンできます。

## GitHub Pages で公開する

1. リポジトリ直下に `index.html` を置く。
2. **Settings → Pages → Build and deployment → Source: _Deploy from a branch_ → Branch: `main` / `(root)` → Save**。
3. 数十秒〜数分で `https://<ユーザー名>.github.io/<リポジトリ名>/` に公開されます。
4. 公開URLを開いて設定 → 「公開リンクをコピー」で**配布用リンク**を取得して配ります。

> 素のURL（`#play=` なし）は管理画面＋既定の盤面が表示されます。特定の構成をプレイヤーに固定したいときは、`#play=…` の公開リンクを使ってください。

## 注意（重要）

抽選ロジックは**クライアント側（この `index.html` 内）**にあります。通常の利用者には設定・確率は見えませんが、開発者ツールを使える人は URL（`#play=`）や JS を解析すれば内容を読み取れます。

**実際の景品が絡む本番**で確実に隠したい・不正対策が必要な場合は、抽選をサーバー側で行い、クライアントには結果だけ返す構成にしてください。
